# Patches that were applied on this kernel, for Referencing

This directory, contains all of the .patches file for applying it in to the source code.

The files that are outside the `_git-am` may be subject to use the `patch` instead. The author advises that util because the Maintainer had some difficulties using `git am` to patch, however, `patch` seemed to work fine.

The files that are in [`_git-am`](_git-am) are the ones that are readily available to patch using `git am`, but please make sure to apply it in the root kernel directory so the git can adjust
